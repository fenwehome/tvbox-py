# 爱奇艺奇异短剧分类设计

## 目标

新增一个独立分类 `奇异短剧`，从现有短剧能力中拆出“短剧优选”内容，作为单独分类暴露给客户端。

## 范围

- 新增首页分类 `奇异短剧`
- 为 `奇异短剧` 提供静态 filters
- 为 `奇异短剧` 增加独立的分类列表加载逻辑
- 保持现有 `短剧` 分类继续可用

## 非目标

- 不重构现有 `短剧`、`漫剧`、`少儿`、`知识` 的动态 filter 逻辑
- 不调整已有分类顺序之外的 UI 协议
- 不修改详情页和播放逻辑，除非新分类的 `vod_id` 需要最小兼容

## 方案

### 分类

在 `Spider._classes()` 中新增：

- `type_name`: `奇异短剧`
- `type_id`: 一个新的独立路由 id，用于与 `topic:short` 区分

推荐使用类似 `topic:qiyi_short` 的新 id，继续沿用现有 `kind:value` 路由格式，减少分支判断成本。

分类顺序要求：

- `奇异短剧` 放在 `百度短剧` 后面
- 不调整其他既有分类的相对顺序

### Filters

`奇异短剧` 不接 `videolib/tag` 动态 filters，而是直接返回静态 filters。

静态 filters 至少包含：

- 排序
- 资费
- 年份
- 类型

实现上应当把 `奇异短剧` 显式排除在动态 filter 映射之外，避免落入 `videolib/tag` 自动抓取。

### 列表数据

新增一个专门的分类加载分支，用于提取“短剧优选”数据。

数据来源策略：

- 复用现有短剧请求链路
- 在返回结果中仅保留属于“短剧优选”的卡片
- 输出结构保持与现有分类列表一致

如果现有短剧接口无法直接按参数筛出“短剧优选”，则在解析层完成筛选；如果接口本身支持更稳定的定位参数，则优先用请求参数限定，而不是依赖弱文本匹配。

### 兼容性

- 现有 `短剧` 分类行为不变
- 新分类的 `vod_id` 继续遵循现有详情页可识别的格式
- 若新分类与 `topic:short` 复用详情逻辑，则应复用已有分支，避免新增 detail 协议

## 错误处理

- 当“短剧优选”源为空时，返回空列表，不影响其他分类
- 若专用提取失败，可按最小可用原则回退到普通短剧提取，但只应在确认不会误混大量非优选内容时启用；默认更安全的行为是返回空列表

## 测试

补充单测覆盖：

- 首页分类包含 `奇异短剧`
- `奇异短剧` 使用静态 filters，而不是动态 `videolib/tag`
- `奇异短剧` 的 `categoryContent()` 走独立分支
- 提取逻辑只返回“短剧优选”结果
- 原 `短剧` 分类仍保持现有行为

## 风险

- “短剧优选”在源数据中的定位方式可能不稳定
- 如果只能依赖页面结构或文案筛选，后续可能随爱奇艺页面调整而失效

当前实现应优先选择最稳定、最显式的识别信号；只有在缺少结构化标记时，才退回文本级识别。
